  • 补充知识

    • retain：ARC之前属性构造器的关键字是retain/copy/assign。 strong和weak是ARC带出来的关键字。retain同现在的strong，就是指针指向值的地址，同时进行引用计数加1。
    • strong：指向并持有该对象，引用计数会加1。引用计数为0才会销毁，可以通过将变量强制赋值 nil 来进行销毁。
    • copy：copy关键字和 strong类似，但copy多用于修饰有可变类型的不可变对象上，如NSString/NSArray/NSDictionary上。当可变对象赋值后，会深拷贝生成新的内存存储，不会产生引用计数增加，这样可以防止可变对象的值改变后对原值造成影响。
    • assign：主要用于修饰基本数据类型，引用计数不会加1，例如NSInteger，CGFloat，存储在栈中，内存有系统管理。如果修饰对象类型，会出现野指针，因为当对象地址引用计数为0时，只是进行了值释放，并不会对指向的地址进行释放，所以会导致野指针存在。
    • weak：指向但是并不持有该对象，引用计数不会加1。在 Runtime 中对该属性进行了相关操作，无需处理，当修饰的变量在地址被释放后，可以自动销毁。（weak修饰的对象在变量地址引用计数为0时，为何被置为 nil呢？ - 查看下篇内容）
    • __unsafe_unretain：类似于 weak ，但是当对象被释放后，指针依然保存着之前的地址，被释放后的地址变为僵尸对象，访问被释放的地址就变成了野指针，所以说它是不安全的。
    • atomic：这个属性是为了保证在多线程的情况下，编译器会自动生成一些互斥加锁的代码，避免该变量的读写不同步的问题。atomic可以保证setter和getter存取的线程安全并不保证整个对象是线程安全的。 比如，声明一个NSMutableArray的原子属性array，此时self.array和self.array = otherArray都是线程安全的。但是，使用[self.array objectAtIndex:index]就不是线程安全的，需要用锁来保证线程安全性。
    • nonatomic ：如果该对象无需考虑多线程的情况，这个属性会让编译器少生成一些互斥代码，可以提高效率。

3. 关键字@property、@synthesize和@dynamic

@property的本质：@property = ivar + getter + setter; 即成员变量和存取方法。

比如定义了属性，默认情况下编译器会帮我们生成getter，setter方法

@property有两个对应的词，分别是@synthesize和@dynamic。
如果@synthesize和@dynamic都没写，那么默认的就是@syntheszie var = _var;

@synthesize的含义就是如果没有手动实现setter方法和getter方法，那么编译器会自动生成这两个方法。
自己平时还有一种用法就是，某个属性在.h中对外是只读的，在.m内部的getter方法中需要判断值是否存在再进行赋值，这种情况用@syntheszie修饰就可以正常使用。 比如:
